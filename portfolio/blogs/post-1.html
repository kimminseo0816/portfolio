Notes on evaluating LLM summaries without a reference

Some practical approaches for judging summary quality when there's no ground-truth summary to compare against — which, in production, is most of the time.

Most summarization papers report ROUGE or BERTScore against a human-written reference summary. In production, you usually don't have one — the whole point is that a human isn't writing a summary for every document. So the question becomes: how do you know if a summary is any good?

Reference-free signals that actually help

A few signals have been useful in practice, roughly in order of how cheap they are to compute:

  • Length and compression ratio relative to the source.
  • Factual consistency checks via natural language inference models, comparing each summary sentence against the source.
  • Coverage — whether the key entities in the source also appear in the summary.
  • Repetition and degeneration checks for n-gram loops.

A minimal evaluation harness

The harness I use runs each candidate summary through three checks and produces a small report: 

def evaluate_summary(source, summary):
    return {
        "compression": len(summary) / len(source),
        "consistency": nli_consistency_score(source, summary),
        "coverage": entity_coverage(source, summary),
    }

None of these checks are perfect on their own, but tracked together over time they catch most regressions — a model that suddenly starts hallucinating facts will show a consistency drop well before anyone notices in a manual review.

The goal isn't a single score. It's a small set of signals that moves together, so when one of them moves alone, you know where to look.

What this doesn't catch

Tone, style, and "is this actually useful to a reader" still need human review. These automated checks are a first filter, not a replacement for periodically reading the outputs yourself.
